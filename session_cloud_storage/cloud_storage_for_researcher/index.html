    About Cloud Storage

    In Short, Cloud Storage is not like a disk because:

    • must be organized for worldwide internet access
    • must hold billions of files from millions of customers
    • must be fast
    • must be ultra-dependable

    Attached Storage = Local Disks

    • Hard Drive attached internal
    • disks and sticks attached via USB
    • "Direct Attached"
    • access using paths
      • Mac: /Volumes/mydatadirive/myproject/myfile.csv
      • Windows: D:\myproject\myfile.csv
    • aside: we are still using terms from the 70s and 80s for storage?

    Network attached Storage

    Systems for sharing files over local network (wifi, ethernet)

    • Disk system in a lab
    • U or company data center
    • great when all on same network
    • research clusters have a disk system shared
    • HPC has ultra high speed network and massive disks
    • network methods windows SMB Linux NSF details aren't important others than your need a network protocol to mimic a direct attached disk and there are just a couple, of those AND they are tricky since disk is shared and networks error prone

    Network attached: sharing with no code re-write

    Huge advantage of network attached is because they look like direct attached to your computer , you don't have to change your code ,

    just the path to the file ( windows and mac examples )

    organized into groups called containers ( buckets on AWS )

    access items individually no file folder hierarchy ( BUT there is a way to fake it if necessary )

    on your computer or a university computer like hpc or our web and data systems , the storage is "near" for fast transfer cloud storage in same region is not fast

    cloud sync: one drive google vs cloud storage

    Cloud attached.

    there are ways to mount cloud storage But must be on the same location, is still slow, cloud storage is not like a regular disk

    Azure has a storage option that is more like network attached but it's more expensive. for small data sets the cost may be worth it. prices are in GB/time. A terabyte for months can get expensive.

    see axure docs for details of these types
